## How tickets work now
1. Member opens a ticket → **My Tickets** / **Submit ticket**
2. Manager assigns on **Ticket Manager** (`/tickets/manage`)
3. Assignee sees it under **My Serving** and can **Answer / update**
4. Managers can **Download backup (JSON)** of the ticket queue

## Help download / re-upload
**Help → Manage** → *Download backup (JSON)* / *Re-upload / import*

## Campuses / branches
**Settings → Campuses / Branches** → enable multi-campus, add campuses, link members.  
Top bar campus switcher appears when multi-campus is enabled.
